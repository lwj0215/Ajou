```
마지막에는 수정된 전체 python 코드를 제시할 것.
아래 코드를 리뷰해줘.
```

## 종합 개선안
- prompt_int()의 숫자 검증 방식이 음수/부호 입력에 취약
- add_todo(), complete_todo(), delete_todo()에서 todos 전역 변수 직접 사용
- complete_todo()에서 이미 완료된 항목은 저장되지 않지만, 상태 변경 흐름이 분기별로 다소 불균형
- load_todos()의 예외 처리에서 FileNotFoundError는 사실상 중복
- load_todos()에서 잘못된 항목을 조용히 버려 데이터 손실이 눈에 띄지 않음
- save_todos() 저장 실패 시 사용자에게만 메시지를 보여주고 복구 전략이 없음
- delete_todo()`는 삭제 확인 단계가 없어 실수 삭제 위험
- 메뉴 선택 입력에서 숫자가 아닌 값도 단순 실패 처리되지만, 재입력 유도가 더 명확하면 좋음
- DATA_FILE` 경로 사용은 비교적 안전하지만, 파일 위치에 대한 설명이 있으면 좋음
- list_todos()`가 출력만 담당하지만 재사용성은 낮음
- input() 호출 시 EOFError` 같은 종료 상황 미처리
- 메뉴 선택이 문자열 비교로만 처리됨
